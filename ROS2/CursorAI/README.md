
Очень разумное решение — перенести настройки перед сменой версии, чтобы всё осталось на своих местах. Переход пройдёт гладко, если следовать этому плану. Он включает четыре шага, но первый из них — самый важный для сохранения данных.

### 💾 Шаг 1: Безопасное резервное копирование (самый важный шаг)

Прежде чем что-либо удалять, необходимо создать резервную копию всех ваших данных. Cursor, как и VS Code, хранит конфигурацию в нескольких директориях. Чтобы ничего не потерять, выполните в терминале следующую команду для создания полной резервной копии:

```bash
mkdir -p ~/cursor_backup
cp -r ~/.config/Cursor ~/.cursor ~/cursor_backup/
```

*   **`~/.config/Cursor/`**: Хранит важные настройки, такие как `settings.json` (настройки редактора), `keybindings.json` (горячие клавиши), а также данные о чатах и состоянии интерфейса.
*   **`~/.cursor/`**: Содержит установленные вами расширения (`extensions`), которые будет очень удобно восстановить целиком, а не скачивать заново.

### 🗑️ Шаг 2: Удаление старой версии Cursor

Теперь, когда ваши данные в безопасности, можно приступить к удалению старой версии, установленной вручную.

1.  **Удалите файлы ручной установки**:
    ```bash
    rm -rf ~/.local/bin/cursor
    ```

2.  **Удалите иконку из меню**, чтобы она не создавала путаницу:
    ```bash
    rm ~/.local/share/applications/cursor.desktop
    ```

### 📦 Шаг 3: Установка новой версии Cursor из .deb-пакета

Теперь можно установить новую официальную версию.

1.  **Установите пакет**. Откройте терминал в папке, куда был скачан файл `cursor_3.2.16_amd64.deb`, и выполните:
    ```bash
    sudo apt install ./cursor_3.2.16_amd64.deb
    ```
    Эта команда не только установит редактор, но и автоматически добавит официальный репозиторий Cursor в вашу систему для будущих обновлений.

### ♻️ Шаг 4: Восстановление настроек и расширений

После завершения установки откройте Cursor, а затем закройте его. Это нужно, чтобы программа создала все необходимые служебные папки. После этого выполните в терминале восстановление данных из резервной копии:

```bash
cp -r ~/cursor_backup/Cursor ~/.config/
cp -r ~/cursor_backup/.cursor ~/
```

После этого можете смело удалять папку с бэкапом, если она больше не нужна:
```bash
rm -rf ~/cursor_backup
```

Теперь при следующем запуске Cursor вы увидите все свои привычные настройки, горячие клавиши и расширения именно в том виде, в котором они были.

Надеюсь, этот план поможет! Если на каком-то из шагов возникнут вопросы, спрашивайте.
